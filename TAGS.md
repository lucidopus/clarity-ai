# Tags Reference

This document lists all available GitHub labels for this repository and provides guidance on when to use them.

## Status Tags

### Active Status
- **urgent** - Tasks that need immediate attention
- **soon** - Tasks to be completed in the near term
- **later** - Planning and design tasks (13 open issues)

### Hold Status
- **idea** - New idea that may or may not ever be implemented
- **hold** - Tasks that are on hold (9 open issues)
- **blocked** - Extra attention is needed (1 open issue)

### Resolution Status
- **cancelled** - This will not be worked on
- **duplicate** - This issue or pull request already exists

## Type Tags

### Issue Classification
- **bug** - Something isn't working (1 open issue)
- **improvement** - This feature needs to be improved (2 open issues)
- **tech-debt** - Technical debt that needs to be addressed (13 open issues)
- **ideas** - Feature ideas and suggestions (1 open issue)
- **documentation** - Improvements or additions to documentation
- **performance** - Performance improvements
- **test** - Testing-related work
- **security** - Security-related issues

### Best Practices
- Always assign at least one status tag and one type tag to each issue
- Update status tags as work progresses
- Use multiple type tags when appropriate (e.g., **bug** + **urgent**)
- Review and update **hold** items regularly to avoid stagnation
